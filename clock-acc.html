Clock Accuracy Algorithm

If observations from 2 clocks are supplied to the user, with each clock's time being offset from the other by a finite amount, how many tries will it take for the user to guess the offset (with relative accuracy)? My Clock Accuracy Algorithm takes in a time offset and a number of trials, hoping to support the Law of Large Numbers.
